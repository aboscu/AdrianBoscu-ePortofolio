Network and Information Security Management

A course about how we can better protect and assess our networks as well as manage particular security concerns.

Learning Outcomes

The module aims to provide students with:

  • An understanding of the fundamentals of computer networks and security.
  • An understanding of the principles of information security management.
  • An understanding of the use and benefits of various monitoring and logging tools.
  • An understanding of how to create and use security vulnerability and assessment tools.
  • An understanding of future network architectures and information security management.
  • The opportunity to reflect on and evaluate their personal development.
  • The ability to present critical arguments for specific actions or outcomes to a diverse audience.

On completion of this module you will be able to:

  1. Identify and analyse security risks and vulnerabilities in IT network systems and determine appropriate methodologies, tools and techniques to manage and/or solve them.
  2. Design and critically appraise computer programs and systems to produce solutions that help manage and audit risk and security issues.
  3. Gather and synthesise information from multiple sources (including internet security alerts & warning sites) to aid in the systematic analysis of security breaches and issues.
  4. Articulate the legal, social, ethical and professional issues faced by information security professionals.

e-Portfolio Activity / Discussion Forum Unit 1

In the paper Compromising a Medical Mannequin (Glisson et al., 2015) we are introduced to different vulnerabilities that IoT devices can have in the medical field as well as what can those vulnerabilities cause when exploited. The report presented the outcome of such vulnerabilities can vary from data leakage, device behaviour rewriting and even physical harm of the patient. The report also demonstrated how easy it is to gain access to some of these devices by using different methods such as Brute Force attacks as well as delian of service. The initial case of brute force attack was achieved by using a BlackTrack CD which is a Linux distribution that contains a variety of exploitation tools for different purposes. In this particular case “airman” and “reaver” was used to gain access to the wireless network. This can be avoided by using an authentication server for wireless access such as 802.1x/WPA-Enterprise instead of preshared key along side a strong wireless authentication protocol. (Spalter, 2021) Another case that was presented in the paper was a brute force attack to also gain access to the wireless network using a different strategy. This can also be mitigated by creating strong authentication profiles as well as a separated authentication server for network access. Finally the denial of service attack was performed to affect device availability using tools from the same linux distribution. Denial of service can be mitigated using a modern firewall as it is the case with Palo Alto NGFW (next generation firewall) that can observe traffic behaviour and stop certain actors from executing such exploitations. (Getting a Handle on DDoS, 2021) As a side note, currently working for a Pharma Company specifically to mitigate vulnerabilities in the OT (operation techonlogy) environment, I can see how important these devices are for the well being of many patients. I personally believe that we should be always ahead of the technological curve when trying to protect such devices from vulnerabilities.

References: Palo Alto Networks. 2021. Getting a Handle on DDoS. [online] Available at: https://www.paloaltonetworks.com/blog/2012/04/getting-a-handle-on-ddos/ [Accessed 5 September 2021]. Spalter, M., 2021. Wi-Fi Brute Force Attacks. [online] Draytek.co.uk. Available at: https://www.draytek.co.uk/information/blog/wifi-brute-force-attacks [Accessed 5 September 2021].
